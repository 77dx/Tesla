# Tesla 测试平台 - 快速开始指南

## 项目启动步骤

### 1. 启动后端服务

```bash
# 进入项目目录
cd /Users/cathy/python_project/Tesla

# 激活虚拟环境（如果有）
source venv/bin/activate

# 启动Django服务器
python manage.py runserver 0.0.0.0:8000
```

### 2. 启动前端服务

```bash
# 进入前端目录
cd /Users/cathy/python_project/Tesla/frontend

# 安装依赖（首次运行）
npm install

# 启动开发服务器
npm run dev
```

### 3. 访问系统

- **前端地址**: http://localhost:5173
- **后端API**: http://localhost:8000/api
- **API文档**: http://localhost:8000/api/schema/swagger/
- **管理后台**: http://localhost:8000/admin

### 4. 默认登录信息

- **用户名**: admin
- **密码**: admin123

## 功能使用流程

### 创建完整测试流程

#### 步骤1: 创建项目
1. 登录系统
2. 点击左侧菜单"项目管理"
3. 点击"➕ 新建项目"按钮
4. 填写项目信息：
   - 项目名称：必填
   - 项目简介：选填
   - 项目地址：选填
5. 点击"确定"保存

#### 步骤2: 创建接口
1. 点击左侧菜单"接口管理"
2. 点击"➕ 新建接口"按钮
3. 填写接口信息：
   - 接口名称：必填
   - 请求方法：GET/POST/PUT/DELETE/PATCH
   - URL：接口地址
   - 所属项目：选择项目
4. 点击"确定"保存

#### 步骤3: 创建用例
1. 点击左侧菜单"用例管理"
2. 点击"➕ 新建用例"按钮
3. 填写用例信息：
   - 用例名称：必填
   - 所属项目：选择项目
   - 关联接口：选择接口
   - 断言规则：必填，JSON格式
4. 点击"确定"保存

#### 步骤4: 创建测试套件
1. 点击左侧菜单"测试套件"
2. 点击"➕ 新建测试套件"按钮
3. 填写套件信息：
   - 套件名称：必填
   - 所属项目：选择项目
   - 运行类型：单次执行/计划任务/Webhook
   - 描述：选填
4. 点击"确定"保存

#### 步骤5: 执行测试
1. 在测试套件列表中找到要执行的套件
2. 点击"▶ 执行"按钮
3. 确认执行
4. 系统会自动跳转到执行结果页面

#### 步骤6: 查看结果
1. 点击左侧菜单"执行结果"
2. 查看执行状态
3. 点击"查看详情"查看详细信息
4. 点击"查看报告"查看Allure测试报告

## 配置说明

### 接口参数配置

接口支持以下参数类型，均使用JSON格式：

```json
// 查询参数 (params)
{
  "key1": "value1",
  "key2": "value2"
}

// 请求头 (headers)
{
  "Content-Type": "application/json",
  "Authorization": "Bearer token"
}

// JSON参数 (json)
{
  "username": "test",
  "password": "123456"
}

// 表单参数 (data)
{
  "field1": "value1",
  "field2": "value2"
}

// Cookies
{
  "session_id": "abc123"
}
```

### 用例断言配置

断言规则使用JSON数组格式：

```json
[
  {"eq": ["$.status", 200]},
  {"contains": ["$.message", "success"]},
  {"gt": ["$.data.count", 0]}
]
```

支持的断言类型：
- `eq`: 等于
- `ne`: 不等于
- `gt`: 大于
- `lt`: 小于
- `gte`: 大于等于
- `lte`: 小于等于
- `contains`: 包含
- `not_contains`: 不包含

### 数据提取配置

使用JSONPath提取响应数据：

```json
{
  "token": "$.data.token",
  "user_id": "$.data.user.id",
  "username": "$.data.user.name"
}
```

### Allure标注配置

## 故障排除

### 问题1: 无法登录
**解决方案**:
1. 检查用户名和密码是否正确
2. 确认后端服务是否正常运行
3. 检查浏览器控制台是否有错误信息

### 问题2: 页面无法加载
**解决方案**:
1. 检查前端服务是否正常运行
2. 清除浏览器缓存
3. 检查网络连接

### 问题3: API请求失败
**解决方案**:
1. 检查Token是否过期（重新登录）
2. 检查后端服务是否正常
3. 查看浏览器Network面板的错误信息

### 问题4: 测试执行失败
**解决方案**:
1. 检查Celery Worker是否运行
2. 查看执行日志
3. 确认测试用例配置正确
