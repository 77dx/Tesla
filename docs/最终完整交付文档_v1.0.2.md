## 🎯 核心功能详解

### 1. 异步执行机制

#### 技术实现
- **任务队列**: Celery
- **消息代理**: Redis
- **执行流程**: 
  1. 用户提交执行请求
  2. 创建RunResult记录
  3. 提交Celery任务
  4. 立即返回result_id
  5. 后台异步执行

#### 代码示例
```python
# Suite.run() 方法
def run(self, endpoint_ids=None, initial_context=None):
    # 创建执行记录
    result = RunResult.objects.create(
        suite=self,
        project=self.project,
        path="todo"
    )
    
    # 创建独立目录
    dir_name = f"result_{result.id}_{int(time.time())}"
    path = Path('upload_yaml') / dir_name
    path.mkdir(parents=True, exist_ok=True)
    
    # 提交异步任务
    from suite.tasks import start_suite_dag
    start_suite_dag.delay(result.id, endpoint_ids, initial_context)
    
    return result
```

### 3. 数据隔离机制

#### 隔离维度
1. **目录隔离**: 每次执行独立目录
2. **文件隔离**: 测试文件、结果文件独立
3. **上下文隔离**: Redis key按result_id隔离
4. **报告隔离**: 每个执行独立的Allure报告

#### 命名规则
```python
# 目录名格式
dir_name = f"result_{result_id}_{timestamp}"

# 示例
result_123_1709876543/
result_124_1709876544/
result_125_1709876545/
```

### 4. DAG依赖调度

#### 功能说明
- 支持接口之间的依赖关系
- 自动构建依赖图（DAG）
- 按依赖顺序分波次执行
- 检测循环依赖

#### 执行流程
```
Wave 1: [Endpoint A, Endpoint B]  # 无依赖，并行执行
  ↓
Wave 2: [Endpoint C]              # 依赖A和B，等待完成后执行
  ↓
Wave 3: [Endpoint D, Endpoint E]  # 依赖C，并行执行
  ↓
Finalize: 合并结果，生成报告
```

### 5. 状态跟踪

#### 状态定义
```python
class RunStatus(models.IntegerChoices):
    Init = 0, "初始化"
    Ready = 1, "准备开始"
    Running = 2, "正在执行"
    Reporting = 3, "正在生成报告"
    Done = 4, "执行完毕"
    Error = -1, "执行出错"
```

#### 状态流转
```
Init → Ready → Running → Reporting → Done
                  ↓
                Error
```

### 执行流程架构
```
User Request
    ↓
Django View (创建RunResult)
    ↓
Celery Task (异步执行)
    ↓
┌─────────────────────────┐
│  start_suite_dag        │ 初始化DAG
└────────┬────────────────┘
         ↓
┌─────────────────────────┐
│  schedule_next_wave     │ 调度执行

         ↓
┌─────────────────────────┐

│  (并行执行多个)          │
└────────┬────────────────┘
         ↓
┌─────────────────────────┐
│  dag_wave_complete      │ 收集结果
└────────┬────────────────┘
         ↓
┌─────────────────────────┐
```
