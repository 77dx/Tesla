# 前后端数据格式对接修复说明

## 修复日期
2026-03-09

## 问题描述

后端使用了统一的响应格式包装器，所有API返回的数据结构为：

```json
{
    "code": 200,
    "message": "ok",
    "result": {
        // 实际数据
    }
}
```

对于列表接口，返回格式为：

```json
{
    "code": 200,
    "message": "ok",
    "result": {
        "page": 1,
        "pageSize": 10,
        "pageCount": 1,
        "itemCount": 4,
        "next": null,
        "previous": null,
        "list": [
            // 列表数据
        ]
    }
}
```

但前端代码错误地直接访问 `res.results` 或 `res.data`，导致无法获取数据。

## 修复方案

### 1. 登录模块修复

**文件**: `frontend/src/stores/user.js`

**修改前**:
```javascript
if (res.data && res.data.token) {
    token.value = res.data.token
}
```

**修改后**:
```javascript
if (res && res.result && res.result.token) {
    token.value = res.result.token
    userInfo.value = res.result
}
```

## 注意事项

1. **响应拦截器**: `frontend/src/api/request.js` 中的响应拦截器返回 `response.data`，所以在业务代码中直接使用 `res` 即可。

2. **可选链操作符**: 使用 `?.` 操作符避免访问 undefined 属性时报错。

3. **兼容性处理**: 在 `fetchUserInfo` 方法中同时兼容了两种响应格式：
   - 带 `result` 包装的格式
   - 直接返回数据的格式

4. **统计数据**: 使用 `itemCount` 字段获取总数，而不是 `count`。

5. **列表数据**: 使用 `result.list` 获取列表数据，而不是 `results`。
