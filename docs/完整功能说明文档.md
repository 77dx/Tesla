# Tesla 测试平台 - 完整功能说明文档

## 项目概述

Tesla 是一个基于 Django + Vue3 的接口自动化测试管理平台，提供项目管理、接口管理、用例管理、测试套件管理和执行结果查看等功能。

## 技术栈

### 后端
- **框架**: Django 4.2
- **REST API**: Django REST Framework
- **认证**: Token Authentication
- **数据库**: SQLite (可配置为 PostgreSQL/MySQL)
- **异步任务**: Celery + Django-Q
- **API文档**: drf-spectacular (Swagger/OpenAPI)

### 前端
- **框架**: Vue 3 (Composition API)
- **路由**: Vue Router 4
- **状态管理**: Pinia
- **HTTP客户端**: Axios
- **构建工具**: Vite

## 认证机制

### Token认证流程
1. 用户通过 `/api/account/profile/login/` 登录
2. 后端验证用户名密码，返回Token
3. 前端将Token存储在localStorage
4. 后续请求在Header中携带: `Authorization: Token {token}`
5. 后端通过Token识别用户身份

### 路由守卫
```javascript
router.beforeEach((to, from, next) => {
  const token = localStorage.getItem('token')
  if (!token && to.path !== '/login') {
    next('/login')  // 未登录跳转到登录页
  } else if (token && to.path === '/login') {
    next('/dashboard')  // 已登录跳转到首页
  } else {
    next()
  }
})
```

## 数据流转

### 请求拦截器
```javascript
api.interceptors.request.use(config => {
  const token = localStorage.getItem('token')
  if (token) {
    config.headers.Authorization = `Token ${token}`
  }
  return config
})
```

### 响应拦截器
```javascript
api.interceptors.response.use(
  response => response.data,
  error => {
    if (error.response?.status === 401) {
      localStorage.removeItem('token')
      window.location.href = '/login'
    }
    return Promise.reject(error)
  }
)
```

## 测试执行流程

### 1. 创建测试数据
- 创建项目
- 创建接口
- 创建用例
- 创建测试套件

### 2. 执行测试
1. 用户点击"执行"按钮
2. 前端调用 `POST /api/suite/suite/{id}/run/`
3. 后端创建RunResult记录
4. 生成独立的执行目录
5. 生成YAML测试文件
6. 提交到Celery异步执行
7. 执行pytest测试
8. 生成Allure报告
9. 更新RunResult状态

### 3. 查看结果
- 在执行结果页面查看状态
- 点击"查看报告"打开Allure报告
- 查看测试日志

## 部署说明

### 后端部署

1. 安装依赖
```bash
pip install -r requirements.txt
```

2. 数据库迁移
```bash
python manage.py makemigrations
python manage.py migrate
```

3. 创建超级用户
```bash
python manage.py createsuperuser
```

4. 启动服务
```bash
python manage.py runserver 0.0.0.0:8000
```

5. 启动Celery Worker (可选)
```bash
celery -A Tesla worker -l info
```

### 前端部署

1. 安装依赖
```bash
cd frontend
npm install
```

2. 开发模式
```bash
npm run dev
```

3. 生产构建
```bash
npm run build
```

4. 配置Nginx (生产环境)
```nginx
server {
    listen 80;
    server_name your-domain.com;
    
    location / {
        root /path/to/frontend/dist;
        try_files $uri $uri/ /index.html;
    }
    
    location /api {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

## 常见问题

### 1. 登录后无法跳转到首页
**解决方案**: 已修复路由配置，登录成功后会自动跳转到 `/dashboard`

### 2. Token过期
**解决方案**: 响应拦截器会自动检测401状态码，清除Token并跳转到登录页

### 3. 测试执行失败
**解决方案**: 
- 检查Celery Worker是否运行
- 查看执行日志
- 确认测试用例配置正确

### 4. 报告无法查看
**解决方案**:
- 确认Allure已安装
- 检查报告路径配置
- 确认静态文件服务配置正确
